# AI Report: интеллектуальная платформа отчётов и ассистентов

**Краткий обзор проекта и последние достижения**

---

## О проекте

**AI Report** — единая платформа для работы с финансовыми отчётами Iridium M2M, биллингом и спутниковой документацией. Система объединяет **базу знаний (KB)**, **RAG-ассистентов** и **веб-интерфейс** на Streamlit: от запроса на естественном языке до готового SQL, от Confluence до структурированной спутниковой базы знаний и диалога с экспертом-помощником.

Ключевые черты:
- **Один вход** — веб-интерфейс (stat.steccom.ru/ai_report): отчёты, загрузка данных, ассистент по биллингу, спутниковый ассистент.
- **Oracle + Qdrant** — оперативные данные в Oracle, векторный поиск и контекст для LLM в Qdrant.
- **Гибкий деплой** — подготовка бандла (`prepare_deployment.sh`), синхронизация на сервер, перезапуск без Docker.

## Ассистент по биллингу

**Суть:** превращает вопрос на русском языке в корректный SQL к Oracle и возвращает результат или подсказку по доработке запроса.

### Что уже есть

- **RAG поверх базы знаний** — примеры вопросов/ответов, описание таблиц и представлений (VIEW) в Qdrant; ассистент подтягивает релевантные примеры и схему перед генерацией SQL.
- **Интеграция с LLM** (GPT-4o через proxyapi.ru) — генерация и правка SQL с учётом контекста и валидации.
- **Безопасность и контроль** — проверка и при необходимости ограничение выполняемых запросов; использование готовых VIEW для отчётов по превышениям, прибыльности, аналитике.
- **Расширение KB** — интерфейс для добавления новых примеров «вопрос → SQL» в базу знаний, чтобы ассистент лучше отвечал на типовые запросы.

### Перспективы и направления развития

- **Голосовой ввод** — озвученный вопрос → текст → SQL (задел по транскрипции уже есть).
- **Более глубокая валидация** — автоматическая проверка по схеме (списки полей, типы), подсказки по индексам и тяжёлым запросам.
- **Шаблоны отчётов** — сохранение «любимых» запросов и быстрый запуск отчёта в один клик.
- **Объяснение результата** — краткое текстовое объяснение: что посчитано, какие фильтры применены.

## Спутниковый ассистент (библиотекарь)

**Суть:** эксперт-помощник по спутниковой документации и пресейлу: Confluence, схемы, спецификации, инструкции. Работает в двух режимах — **инженер** (внутренняя документация, KB) и **абонент** (простые инструкции без служебных деталей).

### Что уже есть

- **Единая спутниковая KB** — контент из Confluence (страницы и вложения) индексируется в ту же коллекцию Qdrant: семантический поиск по инструкциям, схемам, спецификациям.
- **Синхронизация Confluence → KB** — скрипты и веб-интерфейс: по пространству, по списку страниц, слияние (merge), пометка устаревших, зачистка; общий модуль `confluence_sync_runner` для CLI и UI.
- **Парсеры вложений** — PDF, DOCX, XLS/XLSX, изображения (с поддержкой описаний через Vision), draw.io; тип документа учитывается при поиске и подсказках.
- **Диалог с памятью** — история переписки оператора и ассистента сохраняется в SQLite; контекст не теряется при перезагрузке страницы.
- **Спецификации и экспорт** — по итогам диалога ассистент формирует согласованную спецификацию; интерфейс распознаёт строки вида «Наименование — N шт.» и выдаёт:
  - таблицу на экране;
  - выгрузку в **Excel (.xlsx)** и **Word (.docx)** без «слипания» колонок.
- **Draw.io по диалогу** — генерация файла .drawio по текстовой спецификации из переписки (кнопка «Создать draw.io по диалогу»).
- **Рекомендации по наполнению KB** — анализ пространства Confluence (типы вложений, парсеры), подсказки, что синхронизировать в первую очередь.

### Перспективы и направления развития

- **Типизация документов в KB** — явные метаданные (схема сети, список устройств, инструкция, draw.io) для более точного поиска и ответов.
- **Пакетная синхронизация пространств** из веб-интерфейса с предпросмотром и выбором типов контента.
- **Улучшение аннотаций изображений** — массовое описание схем/скриншотов для поиска по содержимому картинок.
- **Интеграция с тикетами/заявками** — передача сформированной спецификации или инструкции в внешние системы (CRM, тикеты поддержки).
- **Мультиязычность** — поддержка запросов и ответов на английском для международных проектов.

## Технологии и интеграции

| Компонент        | Технология / источник        |
|------------------|------------------------------|
| Данные, отчёты   | Oracle Database              |
| Векторный поиск  | Qdrant (коллекция kb_billing) |
| LLM              | GPT-4o (proxyapi.ru), Gemini (спутниковый ассистент) |
| Документация     | Confluence (REST API)        |
| Интерфейс        | Streamlit                    |
| Аутентификация   | SQLite (users.db), bcrypt    |

## Итог

**AI Report** сегодня — это не только отчёты по трафику и биллингу, но и два полноценных ассистента: **биллинг** (вопрос → SQL, расширение KB) и **спутниковый** (документация Confluence, спецификации, draw.io, экспорт в Excel/Word, диалог с сохранением контекста). Дальнейшее развитие идёт в сторону более умной валидации и объяснений (биллинг), типизации документов и автоматизации синхронизации (спутник), а также удобного экспорта и интеграций с внешними системами.
